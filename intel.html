Pyramid of Pain

David Bianco's model ranks how much pain you cause an adversary when you burn each indicator type. Detection at the top of the pyramid forces them to change how they operate — not just a file hash. Click a tier for detail.

TTPs — Tactics, Techniques & Procedures
TOUGH — change how they operate entirely
Behavioral signatures: "beacons every 60s from a child of Office via PowerShell" — forces the adversary to redesign their operation from scratch. Built in Phase 12. Documented in ATT&CK as technique IDs.
Tools — Malware, Implants, Exploits
CHALLENGING — rebuild or replace tooling
The malware binary, C2 framework, exploit. Burned by YARA rules matched on code patterns, JA3 fingerprints, and behavioral capability maps. Covered in Phases 4, 6, 7, and 10.
Network / Host Artifacts
ANNOYING — update the payload, rename the mutex
Mutex names (Global\BV-MTX), registry keys, staging paths (C:\Windows\Temp\~bv.tmp), User-Agent strings, URL patterns. Covered in Phases 8, 9, and 11.
Domain Names
SIMPLE — register a new domain in minutes
C2 domains like cdn-telemetry.xyz used in the BLACKVAULT artifacts. Easy to burn — attackers re-register or use DGA. Hunt by subdomain entropy and query volume (Phase 9).
IP Addresses
EASY — spin up a new VPS in minutes
The C2 IP 185.220.101.47 used throughout BLACKVAULT. Tor exit nodes, rented VPS — trivially replaced. Only useful as a short-term tactical indicator.
Hash Values
TRIVIAL — a single byte change = new hash
SHA-256 of packed.exe, memory-sample.raw, etc. One recompile or UPX pack defeats a hash IOC. Useful only for known-sample lookup; never the primary detection layer.
ATT&CK Techniques Covered in BLACKVAULT
T1566 · T1566.001

Phishing — Spearphishing Attachment

Malicious macro-enabled spreadsheet (.xlsm) delivering the dropper via Office macro execution. The initial access vector in the Phase 12 capstone scenario.

→ Phase 12 capstone · Logs: macro execution from excel.exe
Detection: Sigma: Office process spawning cmd.exe / powershell.exe. Email gateway: block .xlsm/.xlsb attachments from external senders.
T1059.001

Command & Scripting — PowerShell

Download cradle executed via powershell.exe -nop -w hidden -enc [base64]. The decoded command fetches and executes a script directly in memory using IEX (New-Object Net.WebClient).DownloadString.

→ Phase 8 range lab (mem lab) · Event ID 4104 scriptblock logging
Detection: Event 4104 scriptblock logging + 4688 process creation. Sigma: powershell.exe with -enc + DownloadString pattern. Constrained Language Mode as hardening.
T1027 · T1027.002

Obfuscated Files — Software Packing

UPX packing applied to the sample binary (range/malware/packed.exe). Near-8.0 entropy on the .text section, minimal import table, UPX0/UPX1 section names (sometimes stripped). The packing wall that forced static analysis to fail.

→ Phase 4 (detection) · Phase 7 (unpacking)
Detection: YARA: UPX magic bytes and section names. Entropy scan: flag sections > 7.2 bits/byte. Import count < 5 as suspicious threshold.
T1055 · T1055.012

Process Injection — Process Hollowing

Malicious code injected into a suspended legitimate process (svchost.exe started, image swapped). Results in a process with a benign name that doesn't match its on-disk binary — exposed via Volatility's ldrmodules cross-view.

→ Phase 8 (memory forensics) · Phase 11 (rootkit detection)
Detection: Volatility malfind: RWX private region with PE header. Memory ≠ disk image mismatch via ldrmodules. Process parent: svchost from explorer.exe (not services.exe) is wrong.
T1547.001

Boot/Logon Autostart — Registry Run Keys

Persistence via HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run — a "WindowsUpdate" value pointing to C:\Temp\svc.exe. Covered in the rootkit phase as a cross-artifact persistence hunt.

→ Phase 11 range lab (registry tab) · Event ID 7045 service install
Detection: Autoruns.exe / Autoruns for Linux. Sigma: new Run key with path outside System32. Monitor: HKLM\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Run also.
T1543.003

Create or Modify System Process — Windows Service

Malicious service "WinSvc" installed pointing to C:\Temp\svc.exe. Recorded as Event ID 7045 in the System event log — a finding that survives even if the file is deleted.

→ Phase 11 range lab (event log tab)
Detection: Event 7045 with image path outside System32/SysWow64 or in user-writable directories. Sigma: service install + unusual binary path pattern.
T1070.006

Indicator Removal — Timestomping

$STANDARD_INFORMATION timestamps modified by anti-forensic tools. The $FILE_NAME attribute (kernel-maintained) retains original timestamps. A mismatch between $SI and $FN is the exposing inconsistency.

→ Phase 11 range lab (MFT tab)
Detection: MFTECmd output: compare $SI Created vs $FN Created. Suspciously round $SI times (00:00:00.000) are a strong indicator. $FN is kernel-set and cannot be modified by user-mode tools.
T1070.001

Indicator Removal — Clear Windows Event Logs

Event ID 1102 records the deliberate clearing of the Security log. The clearing itself is evidence — an empty log with a 1102 event tells you the adversary was there and cleaned up.

→ Phase 11 range lab (event log tab)
Detection: Alert on Event 1102. Forward logs to a SIEM in real-time so local clearing doesn't destroy the record. Sigma: Event 1102 in Security log.
T1071.001 · T1071.004

Application Layer Protocol — Web + DNS

C2 communication over HTTPS (TLS to 185.220.101.47:443 every ~60s) and DNS exfiltration via hex-encoded subdomains to cdn-telemetry.xyz. Both are covered in the BLACKVAULT pcap and network range labs.

→ Phase 9 (network forensics) · Phase 10 (protocol RE)
Detection: JA3 hash matching known C2 profiles. Beaconing detection: periodic intervals with low jitter. DNS: entropy analysis of subdomain labels. Query volume thresholds.
T1095

Non-Application Layer Protocol — Custom C2

The c2_capture.bin artifact uses a custom binary framing protocol: BE EF magic, 2-byte LE length, 1-byte opcode, XOR-encrypted payload with key 0x5A. No RFC, no known signature — reversed from the wire.

→ Phase 10 range lab (protocol RE)
Detection: Network signature on the BE EF 2-byte magic. Suricata: content rule matching the frame header. JA3 if tunneled inside TLS. Anomaly: unusual port, non-standard port for TLS.
T1048.003

Exfiltration Over Unencrypted Protocol — DNS

Data exfiltrated by encoding hex into subdomain labels: s0-4256.cdn-telemetry.xyz, s1-6c61.cdn-telemetry.xyz, etc. Reassembled from the pcap to decode the stolen data. Covered in the DNS exfil range lab.

→ Phase 9 range lab (pcap / DNS exfil)
Detection: DNS: high query volume to one domain + high subdomain entropy. Threshold: >50 unique subdomains/minute. Zeek: dns.log query length distribution. Block direct DNS egress; route through recursive resolver.
T1486

Data Encrypted for Impact — Ransomware

The Phase 12 capstone scenario ends with CryptEncrypt over *.docx/*.xlsx, files renamed *.vault, and a READ_ME.vault.txt ransom note dropped. The full kill chain from phishing to encryption is reconstructed.

→ Phase 12 capstone scenario
Detection: Canary files in predictable locations (C:\Users\Public\canary.docx). Monitor: mass rename events (bulk extension change). VSS snapshot deletion (Event 4688: vssadmin.exe delete shadows).
Simplified Threat Actor Profiles (educational)

These profiles are simplified educational summaries for pattern recognition and ATT&CK context. They are not intelligence reporting — consult MITRE ATT&CK Groups for authoritative data.

APT28 (Fancy Bear)

Russia · GRU Unit 26165 · Active since ~2004

Russian military intelligence group. Targets government, military, defense, aerospace. Known for X-Agent/EVTX implants, spearphishing with weaponized documents, and living-off-the-land techniques.

  • T1566 Spearphishing attachment
  • T1059.001 PowerShell
  • T1027 Obfuscated files
  • T1071 Application layer C2
  • T1005 Data from local system

Lazarus Group (Hidden Cobra)

DPRK · Reconnaissance General Bureau · Active since ~2009

Prolific North Korean threat actor responsible for the 2014 Sony hack, WannaCry, and major financial institution targeting. Mixes espionage with financial crime — rare combination. Known for extensive custom malware families.

  • T1059.001 / T1059.003 PowerShell + cmd
  • T1055 Process injection
  • T1547.001 Registry persistence
  • T1486 Ransomware (WannaCry)
  • T1041 Exfiltration over C2

FIN7 (Carbanak group)

Financial crime · Eastern Europe · Active since ~2013

Financially-motivated group targeting retail, hospitality, and financial sectors. Known for sophisticated spearphishing campaigns, custom CARBANAK malware, and PoS system compromise. Heavily uses living-off-the-land.

  • T1566.001 Spearphishing + macros
  • T1204 User execution
  • T1546.003 WMI event subscription
  • T1071.001 HTTPS C2
  • T1056 Input capture (keylogging)

Sandworm

Russia · GRU Unit 74455 · Destructive attacks

Russian GRU group behind NotPetya, BlackEnergy, and attacks on Ukrainian power grid. Specializes in destructive operations — wipers rather than espionage. Demonstrates the importance of ICS/OT security.

  • T1078 Valid accounts (credential theft)
  • T1485 Data destruction (wiper)
  • T1489 Service stop (pre-wipe)
  • T1561 Disk wipe
  • T1021 Remote services (lateral movement)

Detection Rule Templates

SIGMA — Office spawning PowerShell (T1059.001 + T1566)

title: Office Process Spawning PowerShell
status: stable
logsource:
    product: windows
    category: process_creation
detection:
    selection:
        ParentImage|endswith:
            - '\WINWORD.EXE'
            - '\EXCEL.EXE'
            - '\POWERPNT.EXE'
        Image|endswith:
            - '\powershell.exe'
            - '\cmd.exe'
            - '\wscript.exe'
    encoded:
        CommandLine|contains: '-enc'
    condition: selection and encoded
level: high
tags:
    - attack.execution
    - attack.t1059.001
    - attack.initial_access
    - attack.t1566

YARA — UPX Packing + Suspicious Imports (T1027.002)

rule Packed_UPX_SuspiciousImports
{
    meta:
        description = "UPX-packed binary with minimal import table"
        technique   = "T1027.002"
    strings:
        $upx0 = "UPX0" ascii
        $upx1 = "UPX1" ascii
        $upx_sig = { 55 50 58 21 }   // UPX! signature
        $mz    = { 4D 5A }            // PE magic
    condition:
        $mz at 0
        and ($upx0 or $upx1 or $upx_sig)
        and pe.number_of_imports < 5
}

title: Security Audit Log Cleared

SIGMA — DNS Exfiltration Detection (T1048.003)

title: Possible DNS Tunneling - High Subdomain Entropy
status: experimental
logsource:
    product: zeek
    service: dns
detection:
    selection:
        qtype_name: 'A'
    filter_legit:
        query|endswith:
            - '.windows.com'
            - '.microsoft.com'
            - '.google.com'
    condition: selection and not filter_legit
    # True positive indicators (evaluate in SIEM):
    # - query length > 40 chars
    # - entropy of first label > 4.0 bits/char
    # - > 30 unique subdomains to same domain in 5 minutes
level: medium
tags:
    - attack.exfiltration
    - attack.t1048.003
Detection engineering workflow: (1) Understand the technique — what exactly does it do? (2) Find the minimal, durable observable — not the hash, but the behavior. (3) Test the rule for false positives before deploying. (4) Tag each rule with ATT&CK IDs so your coverage map stays current. (5) Review quarterly — adversaries iterate.
