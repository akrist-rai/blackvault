DF

Digital Forensics & IR

Evidence acquisition, file system analysis, memory forensics, network forensics, timeline reconstruction — the analyst's full workflow from first alert to final report.

RE

Reverse Engineering

From raw hex to readable logic — assembly fluency, binary format internals, decompilers, crackmes, keygen challenges, and custom protocol decoding.

  • x86/x64 assembly & calling conventions (Phase 2)
  • PE/ELF format deep dive (Phase 3)
  • Ghidra decompiler workflow (Phase 5)
  • Three-level crackme progression
  • Custom C2 protocol reversal (Phase 10)
MA

Malware Analysis

Static triage, behavioral detonation in an isolated lab, unpacking, YARA, capability mapping to ATT&CK — from "what is this file" to a full behavioral report.

  • Static analysis — strings, YARA, capa (Phase 4)
  • Dynamic analysis — strace/Frida (Phase 6)
  • Unpacking & binary patching (Phase 7)
  • Rootkits & persistence artifacts (Phase 11)
  • Ransomware kill-chain capstone (Phase 12)
Everything is synthetic & defensive. Crackmes, simulated pcaps, fabricated binaries, and fictitious log files — built for learning DFIR / RE / MA skills. Nothing here is live malware, and every artifact has a documented educational purpose.

Cross-cutting skill labs

🔐 Cryptanalysis & Hash Cracking

Four encoded flags (Caesar, Base64, XOR, reversed). Three MD5 hashes against a wordlist. Understand why algorithm choice matters.

cat range/crypto/ciphertexts.txt
hashcat -m 0 range/hashcrack/hashes.txt \
        range/hashcrack/wordlist.txt
# all decode to BV{...} flags

🕵️ Steganography & Data Carving

A PNG with hidden data past the IEND marker. A text file with an appended ZIP archive. File carving by signature without a filesystem.

file range/stego/photo.png
strings range/stego/photo.png | grep BV
binwalk range/stego/photo.png
unzip -p range/stego/carve_me.bin

🔓 Crackme Progression

Three compiled ELF crackmes of increasing complexity: hardcoded string → XOR cipher → weighted-checksum keygen.

cd range/crackmes && make
./crackme1 letmein          # ACCESS GRANTED
r2 -A crackme2              # pdc at xor_gate
# crackme3: derive the serial rule

🖥️ Incident Response Logs

SSH brute-force followed by successful admin login. Web shell upload and execution. A full attack kill-chain captured in log format.

grep "Failed password" range/logs/auth.log
grep "POST" range/logs/access.log
# spot: brute force → upload → shell exec

📀 Disk Forensics

A 7 MB DOS-partitioned disk image with an ext4 partition, a deleted file, and a FAT32 partition. Use Sleuth Kit to recover evidence.

mmls range/disk/disk.img
fls -o 2048 range/disk/disk.img
icat -o 2048 range/disk/disk.img 14
# the deleted secret.txt lives at inode 14

🧠 Memory Forensics

512 KB simulated RAM with planted IOCs — a typo-squatted process name, C2 IP, and a Base64-encoded flag. Use strings and pattern matching.

strings -n6 range/memory/memory-sample.raw \
  | grep -Ei 'svch0st|185\.220|exfil'
# find the base64 blob and decode it
base64 -d <<< "QlZ7..."

~120 hours · all tracks

All 12 phases in order. Deep understanding of the connections between phases. The recommended path.

  1. Phases 1–3: foundations
  2. Phases 4–5: static analysis + Ghidra
  3. Phases 6–7: dynamic + unpacking

quick start

Everything runs in the browser or on a Linux machine with standard tools. No special setup required for the Study Console.

# 1. clone the repo
git clone https://github.com/akrist-rai/blackvault
cd blackvault

# 2. open the console in a browser
open index.html    # then click Enter Console

# 3. install tool prerequisites (Linux/macOS)
sudo apt install sleuthkit gdb radare2 binutils upx-ucl tcpdump \
     volatility3 strace ltrace hashcat john binwalk foremost

# 4. build the crackmes
cd range/crackmes && make && cd ../..

# 5. verify the artifacts
sha256sum range/disk/disk.img range/network/capture.pcap \
          range/memory/memory-sample.raw
